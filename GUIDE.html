Domains Tracker — Инструкция

Подробно, по шагам. Для тех, кто не программист.

Каждый раз, когда садишься работать

1Открой Docker Desktop

Найди приложение Docker в Launchpad или через Spotlight (Cmd+Пробел → набери «Docker» → Enter). Подожди, пока в строке меню сверху появится спокойный значок кита — это значит, что Docker запустился и готов к работе. Обычно занимает 20–40 секунд.

Если этот шаг пропустить — все команды ниже будут выдавать ошибку про «Docker daemon». Просто открой Docker Desktop и подожди.
2Запусти приложение командой

Открой Терминал и вставь команду:

dt-start

Эта команда сама сделает резервную копию базы данных и запустит приложение. Подожди, пока в Терминале появится строка Container domains_tracker_backend Started.

3Открой приложение в браузере

Перейди по ссылке:

http://localhost:5051

Работай как обычно: ставь галочки, копируй источники, добавляй ссылки на блокноты.

4Когда закончил — останови приложение

Вернись в Терминал и вставь:

dt-stop

Тоже сделает резервную копию автоматически, затем остановит приложение. Docker Desktop можно после этого закрыть, а можно оставить открытым — как удобнее.

Что означают значки в приложении

✅ — источник обработан (загружен в NotebookLM) 🎧 — аудиопересказ прослушан

Обе метки независимы — можно поставить только одну, обе сразу или ни одной. Фильтры в шапке (нажми «Фильтры ▾») работают так же: можно одновременно включить «обработано: да» и «прослушано: нет», чтобы увидеть всё, что обработано, но ещё не прослушано. Рядом есть кнопка «Сбросить фильтры».

Массовый импорт источников через ИИ

Как добавить сразу много источников по теме

В шапке приложения нажми на значок ☰ (меню) слева от поля поиска → «Импорт источников». Откроется отдельная страница.

Дальше всё происходит в три шага:

1. На странице импорта нажми кнопку «Промт для ИИ» — откроется панель с готовым текстом запроса.
2. В поле вверху панели впиши тему (например «история криптографии» или «теория эволюции»), нажми «Копировать».
3. Вставь промт в любой ИИ-чат (проверено на ChatGPT, Claude, Gemini, Grok, DeepSeek — работает на всех). ИИ сам составит список источников по теме и сразу оформит его в нужном формате JSON.

ИИ выведет результат в блоке кода. Скопируй только сам JSON (от открывающей до закрывающей квадратной скобки), сохрани его в текстовый файл с расширением .json — например через TextEdit, сохранив как обычный текст без форматирования.
Загрузка файла и проверка на дубликаты

На странице импорта нажми «Выбрать файл» (или перетащи файл в зону загрузки) и укажи сохранённый .json. Приложение покажет весь список источников из файла, каждый — с одной из двух меток:

🆕 новый — источника нет в базе, можно смело импортировать.
⚠️ похож на существующий — под источником появится карточка с тем, на что он похож, и процент схожести.

Все «новые» источники отмечены галочкой по умолчанию. Похожие на существующие — сними галочку вручную, если уверен, что это дубликат, или оставь, если убедился, что это разные источники со схожим текстом. Кнопки «Выбрать все» и «Только новые» наверху помогают быстро отметить нужное.

Когда список просмотрен — нажми «Импортировать выбранные». Загруженный файл при этом нигде не сохраняется внутри приложения — после импорта временные данные удаляются автоматически.

Бэкапы (резервные копии)

Бэкап создаётся автоматически

Каждый раз при dt-start и dt-stop приложение само сохраняет копию базы данных в папку ~/projects/domains-tracker/backups/. Хранится 5 последних копий, старые сами удаляются — следить за этим не нужно.

Сделать бэкап вручную (на всякий случай)

Если хочешь подстраховаться перед каким-то рискованным действием — сохрани копию на Рабочий стол:

cp ~/projects/domains-tracker/backend/data/tracker.db ~/Desktop/tracker_SAFE_backup.db
Восстановить из бэкапа

Если что-то пошло не так и нужно вернуть прежнее состояние — выполни по очереди:

dt-stop
cp ~/Desktop/tracker_SAFE_backup.db ~/projects/domains-tracker/backend/data/tracker.db
dt-start

Замени имя файла на нужный бэкап, если восстанавливаешь не из ручного, а из автоматического (например ~/projects/domains-tracker/backups/tracker_1.db).

Реже нужные команды

Проверить, что приложение работает
docker ps

Контейнер domains_tracker_backend должен быть со статусом Up.

Посмотреть логи (если что-то не работает)
cd ~/projects/domains-tracker && docker compose logs
Пересборка после изменений в коде
cd ~/projects/domains-tracker && rm -rf backend/frontend && cp -r frontend backend/frontend && dt-stop && docker compose up --build -d
